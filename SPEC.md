# AI 猜画 - 规格说明

## 1. 项目概述

- **项目名称**: AI 猜画 (Draw & Guess)
- **项目类型**: 单人网页游戏
- **核心功能**: 玩家在画布上绘画，AI 大模型实时分析图像并猜测用户画的是什么
- **目标用户**: 想要与 AI 互动绘画娱乐的用户

### 组件设计

**API Key 设置**
- 输入框带眼睛图标切换显示/隐藏
- 保存到 localStorage
- 显示连接状态指示器

**画布**
- 白色背景，带细微阴影
- 鼠标悬停显示十字准心
- 触摸设备支持

**工具栏**
- 颜色选择: 8种预设颜色 + 自定义取色器
- 笔刷大小: 滑块 1-50px

**AI 猜测面板**
- AI 状态: 等待中/分析中/已完成
- 打字机效果显示猜测结果
- 猜测历史列表 (显示最近5条)

**动画效果**
- 按钮 hover: scale(1.05) + 发光效果
- AI 猜测中: 脉冲动画
- 新猜测出现: 淡入 + 上滑
- 画布绘制: 平滑线条

## 3. 功能规格

### 核心功能

1. **绘画系统**
   - 鼠标/触摸绘画
   - 颜色选择 (预设8色 + 自定义)
   - 笔刷大小调节
   - 撤销功能 (20步)
   - 清空画布

2. **AI 识别系统**
   - 使用 OpenAI Vision API (gpt-4o)
   - 自动识别: 用户停止绘画 2 秒后自动触发
   - 手动触发: 点击"让AI猜猜"按钮
   - 发送画布图像到 API
   - 显示 AI 的猜测结果

3. **设置系统**
   - API Key 输入和保存
   - 识别模式选择: 自动/手动

### 用户交互流程

1. 用户打开页面
2. 输入 OpenAI API Key (保存到本地)
3. 开始绘画
4. AI 自动/手动分析图像
5. 显示猜测结果

### API 调用

**请求**
```javascript
{
  model: "gpt-4o",
  messages: [
    {
      role: "user",
      content: [
        { type: "text", text: "请猜测用户在画布上画了什么。用简洁的中文回答，只需要给出答案，不需要解释。如果不确定，也请给出一个最可能的猜测。" },
        { type: "image_url", image_url: { url: "data:image/png;base64,..." } }
      ]
    }
  ],
  max_tokens: 100
}
```

**响应处理**
- 解析 AI 回复
- 显示打字机效果
- 保存到历史记录

### 边缘情况处理

- 无 API Key: 提示用户设置
- API 错误: 显示错误信息，允许重试
- 网络超时: 10秒后提示失败
- 画布为空: 提示"请先画点什么"
- API 余额不足: 提示检查 API Key
